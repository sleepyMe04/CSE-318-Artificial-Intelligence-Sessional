# Assignment 1 – N-Puzzle Solver using A\* Search

## Problem Statement

Solve the **N-Puzzle** (sliding tile puzzle) optimally using the **A\* Search algorithm**. Given an initial configuration of a k×k board with one blank tile, find the minimum number of moves required to reach the goal state, and print the entire solution path.

The goal state arranges tiles in row-major order with the blank at the bottom-right:
```
1  2  3
4  5  6
7  8  0
```

---

## Algorithm

**A\* Search** is an informed search algorithm that finds the optimal path by minimizing the cost function:

```
f(n) = g(n) + h(n)
```

- `g(n)` — number of moves taken so far (actual cost)
- `h(n)` — heuristic estimate of remaining cost

The implementation uses a **priority queue** (min-heap) as the open list and an **unordered set** (hash map) as the closed list to avoid revisiting states.

### Solvability Check

Before running A\*, the solver checks if the puzzle is solvable using inversion count:
- **Odd k:** solvable if the inversion count is even.
- **Even k:** solvable depends on both inversion count and the blank tile's row from the bottom.

---

## Heuristics Implemented

| Heuristic | Description | Admissible? |
|-----------|-------------|-------------|
| **Hamming Distance** | Number of tiles not in their goal position | ✅ Yes |
| **Manhattan Distance** | Sum of row + column distances for each tile | ✅ Yes |
| **Euclidean Distance** | Sum of straight-line distances for each tile | ✅ Yes |
| **Linear Conflict** | Manhattan + 2 × conflicts in same row/column | ✅ Yes |

**Linear Conflict** is the tightest heuristic and typically yields the best performance (fewest expanded nodes).

## Implementation Notes

- All grid states are dynamically allocated (`int**`) and tracked through linked `Node` pointers to reconstruct the solution path.
- The hash function encodes the board as a concatenated string and uses a polynomial rolling hash.
- Children are generated by sliding the blank tile in all four valid directions.
